Why BYOKit

The hard part isn't calling models. It's the config UX.

Plenty of great libraries call LLM APIs. But every BYOK app re-builds the same provider list, key field, "where do I get a key?" guidance, Keychain plumbing, and connection test by hand. BYOKit ships all of it as polished, themeable SwiftUI — and keeps the model calls behind a swappable protocol.

Features

Everything a key picker needs

Drop in the whole settings center, or compose the pieces you want.

⚡️

One-line settings center

List, add, edit, reorder, delete, and choose the active provider — BYOKSettingsView().

🧭

Per-provider onboarding

Structured "get a key" guide with steps, caveats, and deep links to console, sign-up, docs, and pricing.

🔒

Secure by default

Keys go to the Keychain. The UI masks them. "Test Connection" uses a minimal request.

🗂️

Data-driven catalog

Add a provider by editing providers.json — no UI code. Built-in, optionally overridden via remote OTA.

🔌

Swappable engine

DefaultLLMClient speaks OpenAI, Anthropic, Gemini & Ollama with zero dependencies. Conform to LLMClient for anything else.

📱

Adaptive & themeable

Looks right on iPhone, iPad, and Mac. Match the host app with one theme modifier.

Quick Start

Two steps to a full BYOK screen

Add the package, then render the settings view. That's it.

1 · Add the package

// Package.swift
.package(url: "https://github.com/everettjf/BYOKit", from: "1.0.0")

2 · Show the settings center

import BYOKit

struct SettingsScreen: View {
    var body: some View {
        BYOKSettingsView()
            .environmentObject(store)              // a ConfigurationStore
            .byokClient(DefaultLLMClient())     // or your own LLMClient
    }
}

Send a request with the active config

if let config = store.activeConfiguration,
   let provider = await ProviderRegistry.shared.provider(config.providerID) {
    let resolved = ResolvedConfiguration(
        provider: provider,
        configuration: config,
        apiKey: store.apiKey(for: config.id)
    )
    let response = try await DefaultLLMClient().complete(.text("Hello"), with: resolved)
    print(response.text)
}
